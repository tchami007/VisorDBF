# Requirements: VisorDBF v1.1 — Mejoras Técnicas

**Defined:** 2026-07-10
**Core Value:** Un usuario puede abrir cualquier archivo DBF, ver su contenido inmediatamente y exportarlo a TXT con la configuracion exacta que necesita — en menos de tres clicks desde el inicio.

**Source:** Auditorías de code smells (`dotnet-csharp-code-smells`) y rendimiento (`analyzing-dotnet-performance`). Plan completo en `docs/MEJORAS_MILESTONE_V1.md`.

## v1 Requirements

### Correcciones Críticas

- [ ] **CRIT-01**: Memory leak en RelayCommand — desuscripción de CommandManager.RequerySuggested. Ver §1.1
- [ ] **CRIT-02**: Empty catch silencioso en DbfReaderService.ParseDateValue. Ver §1.2
- [ ] **CRIT-03**: Missing innerException al wrappear excepciones en SybaseExportService. Ver §1.3
- [ ] **CRIT-04**: Case-sensitive dictionary lookup en Sybase transfer. Ver §1.4

### Correcciones de Media Severidad

- [ ] **MED-01**: Missing GC.SuppressFinalize en FileLogger. Ver §2.1
- [ ] **MED-02**: Log-and-swallow sin feedback al usuario en ProbeFirstRecordAsync. Ver §2.2
- [ ] **MED-03**: Dead code — eliminar DbfDataReader y MapColumnType. Ver §2.3
- [ ] **MED-04**: Connection string Sybase duplicada en 4 lugares. Ver §2.4

### Refactorización de Diseño

- [x] **REF-01**: Extraer lógica de SybaseExportService (837 lines → 4 clases). Ver §3.1
- [x] **REF-02**: Extraer lógica de MainViewModel (674 lines). Ver §3.2
- [x] **REF-03**: Partir BuildConvertFunction en métodos por tipo. Ver §3.3
- [x] **REF-04**: Reducir parámetros de BuildLine (6 → objeto ExportLineContext). Ver §3.4

### Optimizaciones de Rendimiento .NET

- [ ] **PERF-01**: Sellar 21 clases concretas no selladas (72%). Ver §4.1
- [ ] **PERF-02**: Eliminar LINQ closure allocation por registro en BuildLine. Ver §4.2
- [ ] **PERF-03**: FrozenDictionary para diccionarios estáticos de solo lectura. Ver §4.3
- [ ] **PERF-04**: Capacidad inicial para lista de registros en DbfReaderService. Ver §4.4
- [ ] **PERF-05**: Eliminar Encoding.RegisterProvider redundante en métodos. Ver §4.5
- [ ] **PERF-06**: Optimizar SanitizeFileName (LINQ Contains en char[]). Ver §4.6

## v2 Requirements (Deferred)

- **PERF-07**: Per-call List/Dictionary allocations en cold paths — no accionable sin profiling. Ver §4.7
- Exportacion a formatos adicionales (CSV, JSON, Excel)
- Seleccion de columnas y filtrado
- Modo CLI

## Out of Scope

| Feature | Reason |
|---------|--------|
| Edición o modificación de registros DBF | Herramienta de solo lectura |
| Exportación a formatos distintos de TXT | V2+ |
| Soporte multi-archivo o merge de tablas | Complejidad no justificada |
| Conexión a bases de datos remotas | Fuera del dominio de archivos locales |
| Modo CLI | Interfaz gráfica es el canal principal |
